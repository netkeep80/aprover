## Проекция 9: 4-значная логика как структура запросов

### Онтологический статус четырёх значений

В МТС четыре базовых значения — это **не истинностные константы**, а **четыре фундаментальных запроса по форме** к связям вокруг акорня ∞:

| Значение | Форма связи | Онтологический статус запроса |
|----------|-------------|-------------------------------|
| `[`      | `♂∞`        | «Найти все связи с самозамкнутым началом, указывающим на ∞» |
| `]`      | `∞♀`        | «Найти все связи с самозамкнутым концом, исходящим из ∞» |
| `1`      | `♂∞ ⟼ ∞♀`   | «Найти все связи с началом формы `[` и концом формы `]`» |
| `0`      | `∞♀ ⟼ ♂∞`   | «Найти все связи с началом формы `]` и концом формы `[`» |

Ключевой принцип: каждое значение — это **паттерн различения**, а не абстрактное «состояние». Запрос `[` не «обозначает начало» — он *есть* акт различения формы самозамыкания начала.

### Дуальность как структурная симметрия

Оператор инверсии `!` реализует **структурную симметрию асети связей**:

```
!([) = ]    // Инверсия начала → конец
!(]) = [    // Инверсия конца → начало
!(1) = 0    // Инверсия наличия → отсутствие
!(0) = 1    // Инверсия отсутствия → наличие
```

Это не «логическое отрицание» в классическом смысле. Оператор `!` меняет **направленность структуры запроса**:
```
!(a ⟼ b) = (b ⟼ a)
```
Если запрос `1` ищет связи формы «начало→конец», то `!1` ищет связи формы «конец→начало» — то есть `0`.

### Почему это не сводится к булевой логике

| Критерий | Классическая булева логика | Логика МТС |
|----------|----------------------------|------------|
| **Природа значений** | Абстрактные истинностные состояния | Конкретные паттерны форм связей |
| **Семантика** | Внешняя (интерпретация в моделях) | Внутренняя (резолюция запросов в асети) |
| **Отрицание** | Меняет истинностное значение | Меняет направленность структуры |
| **Контекст** | Значения независимы от контекста | Значения определяются через отношения в асети |
| **Основание** | Принятое соглашение (истина/ложь) | Онтологический факт (наличие/отсутствие направленной связи) |

Критическое отличие: в булевой логике `¬true = false` — это соглашение о смене истинностного значения. В МТС `!1 = 0` — это **структурный факт**: инверсия направленности связи `♂∞ ⟼ ∞♀` порождает связь `∞♀ ⟼ ♂∞`, которая по определению есть `0`.

### Архитектура запросов в асети связей

Четыре значения образуют **замкнутую структуру запросов**, отражающую архитектуру асети:

```
        ∞
       / \
      /   \
     [     ]   ← запросы к полюсам акорня
      \   /
       \ /
        |
     1 / \ 0   ← запросы к связям между полюсами
```

- `[` и `]` — запросы к **полюсам** асети (экстремальные точки различения)
- `1` и `0` — запросы к **связям между полюсами** (направленные отношения)

Эта структура не может быть сведена к двум значениям, потому что:
1. Полюса `[` и `]` онтологически первичны относительно связей `1` и `0`
2. Удаление полюсов разрушает возможность различения направленности
3. Запросы к полюсам и запросы к связям — разные онтологические акты

### Практическая реализация логических операций

В МТС логические операции реализуются как **композиция запросов**:

```
И (конъюнкция):   a ∧ b  :=  (a -> (b -> 0)) -> 0
ИЛИ (дизъюнкция): a ∨ b  :=  !(!a ∧ !b)
```

Но ключевой момент: результат операции — не «истинностное значение», а **новый запрос по форме**, который затем резолвируется в асети. Например:
```
1 ∧ 1 = ((1 -> (1 -> 0)) -> 0)
      = ((1 -> 0) -> 0)   // так как (1 -> 0) = 0
      = (0 -> 0)          // запрос к связи формы «отсутствие→отсутствие»
      = 1                 // резолюция даёт связь формы 1
```

### Отличие от других 4-значных логик

В отличие от логик Клини или Белнапа (где 4 значения кодируют «истина/ложь/неизвестно/противоречие»), логика МТС:

- Не имеет «неизвестного» состояния — запрос всегда резолвируется в асети
- Не имеет «противоречия» — структурная тождественность гарантирует непротиворечивость
- Значения не ранжированы по «степени истинности» — они ортогональны как структурные паттерны

### Заключение: логика как структура запросов

4-значная логика МТС — это **не расширение булевой логики**, а **радикальная переонтологизация логики**:

> Логика есть структура запросов по форме в асети связей, а не система истинностных значений.

Четыре значения отражают **минимально необходимую и достаточную структуру** для различения:
1. Начала различения (`[`)
2. Конца различения (`]`)
3. Направленного различения (`1`)
4. Инвертированного различения (`0`)

Эта структура замкнута относительно операции инверсии `!` и образует **онтологический базис** для всех последующих логических конструкций. Классическая булева логика возникает как **фрагмент** этой структуры при фиксации контекста (полюсов `[` и `]`), но сама по себе не является фундаментом — она производна от структуры запросов в асети связей.
