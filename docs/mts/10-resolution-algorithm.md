### Алгоритм резолюции запросов по форме в асети связей

В Метатеории связей резолюция запроса — это не вычисление или построение, а **акт различения форм** в онтологически данном океане связей (асети). Связи существуют априори; знаки лишь задают паттерны для их различения. Алгоритм реализует трёхфазный процесс, где каждая фаза — внутренний акт связывания, а не внешняя операция над текстом.

---

#### Фаза 1: Лексическая — знак как запрос по форме

Каждый знак формальной нотации есть уже связь, определённая своим паттерном различения:

| Знак | Запрос по форме | Онтологический статус |
|------|-----------------|----------------------|
| `♂` | «Найти все связи формы `y = y → x` для данного `x`» | Префиксный конструктор запроса самозамыкания начала |
| `∞` | «Найти все связи формы `x = x → x`» | Запрос к форме полного самозамыкания |
| `]` | «Найти все связи формы `∞ → ∞♀`» | Абит конца контекста (самозамкнутый конец акорня) |

**Важно:** знак не «обозначает» связь — он *есть* связь-запрос, чья форма задаёт критерий отбора в асети.

---

#### Фаза 2: Синтаксическая — композиция запросов

Правила группировки — онтологические принципы композиции запросов, а не метаправила:

1. `♂` — правоассоциативный префикс: `♂∞]` группируется как `(♂∞) ]`
2. Отсутствие явного оператора связи между `(♂∞)` и `]` подразумевает левоассоциативную связь `→`
3. Итоговая форма запроса: `(♂∞) → ]`

Композиция запросов создаёт новый паттерн различения:
```
запрос_(♂∞) → запрос_]  ⇝  запрос_«найти связь с началом формы (♂∞) и концом формы ]»
```

---

#### Фаза 3: Семантическая — резолюция в асети

Резолюция — поиск в асети связей по заданному паттерну. Алгоритм **не строит** связи — он находит их в онтологически данном океане.

##### Пошаговая резолюция запроса `♂∞]`:

**Шаг 1. Резолюция подзапроса `∞`:**
```
Запрос: «Найти связи формы x = x → x»
Результат: связь ∞ (гарантировано единственность по аксиоме ∞)
```

**Шаг 2. Резолюция подзапроса `♂∞`:**
```
Запрос: «Найти связи формы y = y → ∞»
Результат: связь ♂∞, удовлетворяющая ♂∞ = ♂∞ → ∞
(гарантировано существование по аксиоме самозамыкания начала)
```

**Шаг 3. Резолюция подзапроса `]`:**
```
Запрос: «Найти связи формы ∞ → ∞♀»
Результат: связь ∞♀ (по определению абита конца: ] : ∞♀)
```

**Шаг 4. Резолюция составного запроса `(♂∞) → ]`:**
```
Запрос: «Найти связь с началом формы ♂∞ и концом формы ∞♀»
Результат: связь (♂∞ → ∞♀)
(гарантировано существование по аксиоме связи: ∀r∀e ∃(r → e))
```

**Итог резолюции:**
```
♂∞]  ⇝  (♂∞ → ∞♀)  ⇝  1
```
(поскольку `1 : ♂∞ → ∞♀` по аксиоме единицы смысла)

---

#### Ключевые онтологические принципы

1. **Априорное существование связей:**  
   Асеть связей — онтологически данное целое. Связи не «создаются» запросом — они существуют до и независимо от различения. Запрос лишь выделяет подмножество по форме.

2. **Знак = связь-запрос:**  
   Символ `♂` — не метаобозначение, а связь, чья форма есть паттерн самозамыкания начала. Нет разрыва «знак ↔ связь» — есть только связи разной степени абстракции.

3. **Резолюция ≠ вычисление:**  
   Алгоритм не «вычисляет» `(♂∞ → ∞♀)`, а находит её в асети как связь, удовлетворяющую форме запроса. Это различение, а не конструирование.

4. **Единственность резолюции:**  
   Для корректных запросов (с соблюдением *guarded recursion*) резолюция даёт единственную связь — гарантируется аксиомами тождественности и существования.

---

#### Почему это фундаментальнее классических пруверов

В Rocq/Lean4:
- Нотация — внешний синтаксический сахар
- Ядро работает с абстрактными синтаксическими деревьями
- Семантика задаётся внешней моделью

В МТС:
- Нотация — внутренний язык запросов по форме
- Синтаксический разбор — акт связывания в самой системе
- Семантика = асеть связей (онтологически данное целое, не внешняя модель)

Резолюция запроса `♂∞]` — это не преобразование текста в терм, а **онтологический акт различения формы** в асете, где связи существуют как неделимая ткань бытия. Знак работает не как ссылка, а как паттерн, выделяющий устойчивую структуру в потоке различений.
