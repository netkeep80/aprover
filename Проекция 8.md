### Проекция 8: Абиты как запросы к формам

#### 1. Абиты — не примитивы, а запросы к формам

В МТС абиты **не являются** элементарными «строительными блоками» вроде битов. Это **запросы по форме** к четырём базовым связям, возникающим в непосредственной окрестности акорня ∞:

| Абит | Запрос к форме | Онтологический смысл |
|------|----------------|----------------------|
| `[`  | `♂∞`           | Начальный смысл: связь, у которой начало самозамкнуто на себя, а конец — акорень |
| `]`  | `∞♀`           | Конечный смысл: связь, у которой начало — акорень, а конец самозамкнут на себя |
| `1`  | `♂∞ ⟼ ∞♀`      | Наличие направленной связи (истина как факт различения) |
| `0`  | `∞♀ ⟼ ♂∞`      | Отсутствие/инверсия связи (ложь как отсутствие различения) |

Ключевой принцип: абит `[` не «кодирует» начало — он **есть** запрос к форме `♂∞`. Когда этот запрос резолвируется в асети связей, результатом становится сама связь `♂∞`. То же для остальных абитов.

#### 2. Почему ∞ не является абитом

Акорень ∞ обладает принципиально иным онтологическим статусом:

| Свойство | Абиты (`[`, `]`, `1`, `0`) | Акорень ∞ |
|----------|----------------------------|-----------|
| Природа | Запросы к **частичным** формам самозамыкания | Запрос к **полному** самозамыканию |
| Форма | `♂∞`, `∞♀`, `♂∞ ⟼ ∞♀`, `∞♀ ⟼ ♂∞` | `x = x ⟼ x` |
| Роль в асети | Связи **вокруг** акорня (соседи первого порядка) | Центр асети, условие возможности всех других связей |
| Сериализация | Входят в состав ачисел как элементы последовательности | **Не сериализуется** — служит неявной точкой начала ачисла |

∞ не может быть абитом, потому что:
- Абиты описывают **роли** в акте связывания (начало, конец, наличие, отсутствие)
- ∞ — это **условие возможности** самого различения ролей
- Включение ∞ в ачисло создало бы онтологический парадокс: точка отсчёта стала бы частью отсчитываемого

Поэтому в ачислах ∞ присутствует **неявно** как маркер начала последовательности: любое ачисло `α` интерпретируется как `∞α` на уровне резолюции запросов.

#### 3. Резолюция `[]` → ∞ через композицию запросов

Ачисло `[]` — это композиция двух запросов:

```
[  → запрос к форме ♂∞  → резолюция: связь ♂∞
]  → запрос к форме ∞♀  → резолюция: связь ∞♀
```

Композиция происходит через акт связывания:

```
[]  ≡  [ ⟼ ]          // ачисло как последовательность запросов
    ≡  (♂∞) ⟼ (∞♀)    // подстановка резолюций запросов
    ≡  ♂∞ ⟼ ∞♀        // упрощение скобок
```

Но это **не** даёт ∞ напрямую. Чтобы получить ∞, нужна **слияние рекурсий**:

```
По аксиоме ♂:  ♂∞ ≡ ♂∞ ⟼ ∞
По аксиоме ♀:  ∞♀ ≡ ∞ ⟼ ∞♀

Композиция:    (♂∞) ⟼ (∞♀) 
             ≡ (♂∞ ⟼ ∞) ⟼ (∞ ⟼ ∞♀)

Но из теоремы о слиянии рекурсий:  ♂♀ ≡ ∞

Следовательно, композиция форм ♂∞ и ∞♀ через акт связывания
восстанавливает полное самозамыкание:
             (♂∞ ⟼ ∞♀) при замыкании контекста [] даёт ∞
```

Строгое доказательство:

1. `[` резолвируется в `♂∞`
2. `]` резолвируется в `∞♀`
3. Последовательность `[]` интерпретируется как запрос к связи между результатами (1) и (2): `(♂∞) ⟼ (∞♀)`
4. По аксиоме начала смысла: `♂∞ ≡ ♂∞ ⟼ ∞`
5. По аксиоме конца смысла: `∞♀ ≡ ∞ ⟼ ∞♀`
6. Подстановка: `(♂∞ ⟼ ∞) ⟼ (∞ ⟼ ∞♀)`
7. По теореме о слиянии рекурсий (`♂♀ ≡ ∞`) и свойству ассоциативности акорня: эта композиция резолвируется в `∞`

Таким образом, `[]` — это **минимальный запрос**, чья резолюция восстанавливает акорень. Это не «кодирование» ∞, а **восстановление** его через композицию двух частичных самозамыканий.

#### 4. Структура асети вокруг акорня

Визуализация четырёх базовых форм:

```
        ♂∞          // абит [
         ↓
         ∞ ←─────── ∞   // акорень (не абит!)
         ↓
        ∞♀          // абит ]

♂∞ ⟼ ∞♀  // абит 1 (стрелка вправо)
∞♀ ⟼ ♂∞  // абит 0 (стрелка влево)
```

Эти четыре связи — **единственные** непосредственные соседи акорня в асети. Все остальные связи строятся через композицию этих форм. Поэтому именно они становятся абитами — не по соглашению, а по онтологической необходимости.

#### 5. Практическая сериализация

При сериализации связи в ачисло:

1. Связь разлагается на последовательность элементарных актов различения
2. Каждый акт сопоставляется одной из четырёх базовых форм вокруг ∞
3. Форма заменяется соответствующим абитом
4. Результат — последовательность абитов, **неявно начинающаяся с ∞**

Пример: связь `1` (единица смысла) сериализуется как `1`, но интерпретируется как `∞1`, где:
- `∞` — неявная точка начала
- `1` — запрос к форме `♂∞ ⟼ ∞♀`

Ачисло `[]` интерпретируется как `∞[]`, где композиция запросов `[` и `]` резолвируется обратно в ∞, замыкая контекст.

#### 6. Онтологическая чистота

Подход сохраняет онтологическую чистоту МТС:

- **Нет внешних примитивов**: абиты не вводятся как «данность» — они выводятся из форм связей вокруг ∞
- **Нет разрыва «знак ↔ связь»**: абит `[` не «обозначает» начало — он есть запрос к форме `♂∞`, которая сама является связью
- **Замкнутость системы**: асети достаточно для резолюции всех запросов — нет необходимости во внешних моделях или интерпретациях
- **Конструктивность**: абиты не описывают связи — они конструируют их через акты запроса и резолюции

Абиты — это **язык запросов к асети**, а не «кодировка» связей. Они работают как фильтры, выделяющие из океана связей те структуры, которые соответствуют четырём базовым формам вокруг акорня. Это делает ачисла не «представлением данных», а **протоколом различения форм** в ассоциативной памяти.
